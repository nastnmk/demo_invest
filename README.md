# Инвест-симулятор (frontend)

Клиент для демо-портфеля: котировки и свечи через backend API (данные с **MOEX**), покупка/продажа, аналитика, коэффициент Шарпа, стресс-тесты по сценариям с сервера.

## Требования

- [Node.js](https://nodejs.org/) 18+  
- Запущенный backend (по умолчанию ожидается `http://213.155.14.44:8000` или свой URL)

## Запуск локально

```bash
npm install
npm run dev
```

Откройте в браузере адрес, который выведет Vite (обычно `http://localhost:5173`).

### Переменные окружения

Скопируйте `.env.example` в `.env.local` и при необходимости измените:

| Переменная | Назначение |
|------------|------------|
| `VITE_API_BASE_URL` | Прямой URL API (для production-сборки). В dev часто **пусто** — запросы идут через прокси Vite. |
| `VITE_API_PROXY_TARGET` | Куда проксировать `/api` в режиме разработки (`vite.config`). |

## Сборка

```bash
npm run build
```

Статическая разметка попадёт в `dist/`. Для работы с API задайте `VITE_API_BASE_URL` на ваш backend.

## Структура проекта

- `src/api/` — HTTP-клиент, запросы к REST API (`moex.ts`: портфель, цены, стресс-сценарии, аналитика).
- `src/components/` — экраны и виджеты (маркет, портфель, риск-тест, сообщество).
- `src/auth/` — вход и токены.

## Соответствие чек-листу (этап 3)

| Раздел | Реализация на фронте |
|--------|----------------------|
| Данные MOEX | Цены и свечи приходят с backend, не захардкожены в коде. |
| Ошибки API | Сообщения при недоступности сети/сервера (`ApiError`, баннер в приложении). |
| Портфель | Покупка/продажа с проверкой баланса и наличия позиций; сообщения при отказе. |
| Стресс-сценарии | Сценарии и коэффициенты с API (`/stress-scenarios/full`); итог в ₽ и %; пояснения по бумагам; подсветка худших/лучших строк. |
| Шарп | Отображается из метрик API и/или упрощённый расчёт в аналитике; обновляется при обновлении портфеля. |
| Репозиторий | Для зачёта по пункту «публичный GitHub» выложите проект в открытый репозиторий самостоятельно. |

**Минимум два сценария** для максимального балла по стресс-тестам задаются на **backend** (БД/конфиг); фронт отображает все пришедшие сценарии.
